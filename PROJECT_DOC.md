# 项目优化清单 (v0.9.0)

本项目旨在提升桌面应用的用户体验、代码结构规范性及 UI 交互感。以下是已完成及后续待处理的优化任务，将按顺序执行。

## 已完成任务
1. **版本升级**：全局版本号提升至 `v0.9.0`。
2. **应用架构重构**：
   - 移除 `App.css`，采用 Tailwind v4 样式体系。
   - 实现路由重定向：`/` -> `/overview`。
   - 引入侧边栏导航，集成 `Overview` (概览)、`Video`、`Images`、`Settings`。
   - 使用 GSAP 实现各页面及组件的平滑入场动画。
3. **性能优化**：
   - 使用 `useCallback` 稳定核心处理函数。
   - 使用 `useRef` 优化状态同步，避免不必要的 `useEffect` 重绑定。
4. **UI 组件标准化**：
   - 重构 `shadcn/ui` 组件以遵循 v4 语义规范（使用 `data-slot`、`size-*`、`data-icon`）。
   - 优化 `Progress` 组件交互动画。
   - 增强视频任务卡片处理状态的视觉反馈。

## 待处理任务（按顺序执行）

### 任务 1: 逻辑代码重构 (抽离通用 Hook)
- **目标**：将视频和图片路由中重复的目录扫描和任务管理逻辑抽离为 `useTaskScanning` 等公共 Hook。
- **目的**：减少代码冗余，提高可维护性。

### 任务 2: 交互体验优化 (拖拽上传区域)
- **目标**：为所有支持文件拖拽的页面（视频/图片）添加拖拽上传区域的视觉效果。
- **细节**：增加虚线框、Hover 放大反馈、全局拖拽覆盖区域。

### 任务 3: 任务反馈增强 (日志面板)
- **目标**：实现任务处理失败时的详细错误反馈。
- **细节**：在任务卡片中集成错误图标，点击可查看详细堆栈信息（使用 Dialog 组件展示）。
